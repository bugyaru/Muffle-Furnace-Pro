## 🧠 Общая архитектура

ESP32-C3 выступает **единым мозгом** системы:

- 📡 **Сеть**: работает в режиме точки доступа (`Furnace_Config`) или подключается к вашему роутеру (STA).
- 🌐 **Веб-сервер**: отдаёт `index.html`, `chart.js`, CSS и принимает API-запросы.
- 🔄 **WebSocket**: отправляет данные в браузер каждые 2 секунды без перезагрузки страницы.
- 🌡️ **Датчик**: MAX31855 читает термопару по SPI.
- ⚡ **Управление**: PID-регулятор рассчитывает мощность, а `controlSSR()` включает/выключает твердотельное реле.
- 💾 **Хранение**: LittleFS хранит веб-файлы, программы нагрева и логи.

---

## 🔄 Как работает основной цикл (`loop()`)

Каждые `1000 мс` (1 сек) происходит следующее:

1. **Чтение температуры** → `readTemperature()`
2. **PID-расчёт** → `myPID.Compute()` (если включён нагрев)
3. **Управление реле** → `controlSSR()` (включает SSR на нужную часть 2-секундного окна)
4. **Аварийная проверка** → если `input > 1050.0` → мгновенное отключение SSR
5. **Логирование** → запись в CSV каждые 10 сек
6. **Отправка в браузер** → `sendWsStatus()` каждые 2 сек через WebSocket
7. **Обновление OLED** → каждые 500 мс
8. `server.handleClient()` и `webSocket.loop()` обрабатывают входящие HTTP/WS запросы

## 🌡️ Чтение датчика и заглушка `1050.1°C`

```cpp
float readTemperature() {
  float t = thermocouple.readCelsius();
  if (isnan(t) || t > 1200 || t < -100) {
    // Если обрыв, замыкание или нет ответа → аварийная температура
    input = 1050.1; 
    return input;
  }
  input = t;
  return input;
}
```
- `1050.1` выбрано специально: оно **строго больше** `EMERGENCY_TEMP (1050.0)`, поэтому условие `if (input > EMERGENCY_TEMP)` мгновенно срабатывает и отключает нагрев.
- Это безопасный fallback: даже если датчик отвалится, печь не уйдёт в перегрев.

## ⚙️ PID и управление нагревом

- `myPID.Compute()` вычисляет `output` от `0` до `255`.
- `controlSSR()` преобразует это в **широтно-импульсную модуляцию** с окном `2000 мс`:
    - Если `output = 128` (50%) → SSR включён 1 сек, выключен 1 сек.
    - Если `output = 0` → SSR всегда выключен.
    - Если `output = 255` → SSR всегда включён.

---

## 📜 Как работают программы нагрева

1. Файл `programs.json` хранит массив: `[{"name":"Отжиг","prg":"600,30;850,45"}]`
2. `parseProgramString()` разбивает строку на шаги: `temp → 600, min → 30`
3. `runProgramStep()` линейно интерполирует уставку между шагами
```cpp
setpoint = prevTemp + (targetTemp - prevTemp) * progress
```
1. После завершения всех шагов → программа останавливается, нагрев отключается.

```cpp
if (input > EMERGENCY_TEMP) { 
  digitalWrite(SSR_PIN, LOW);

  programRunning = false;
```

- Срабатывает **независимо от режима** (ручной или программа).
- `input` становится `1050.1` при ошибке датчика → защита активируется автоматически.

## 🔍 Что такое PID в вашей печи?

PID (**P**roportional-**I**ntegral-**D**erivative) — это алгоритм, который **рассчитывает нужную мощность нагрева**, чтобы температура плавно достигла заданной уставки (`setpoint`) и держалась на ней без раскачки и перелётов.

В вашем скетче он не управляет температурой напрямую, а управляет **скважностью включения ТЭНа** через твердотельное реле (SSR).

---

## ⚙️ Как работают три компоненты

|Компонент|За что отвечает|Пример в печи|Что будет, если параметр слишком большой|
|---|---|---|---|
|**Kp** (Пропорциональная)|Реагирует на **текущую ошибку**: `ошибка = setpoint - input`|Если до цели 200°C, даёт полную мощность. Если осталось 10°C, снижает мощность пропорционально.|Печь будет постоянно колебаться вокруг уставки (перелёт → недолёт).|
|**Ki** (Интегральная)|Накапливает ошибку **во времени**. Устраняет "хвост" недогрева.|Если печь долго не может дойти до 900°C (теряет тепло), Ki постепенно добавляет мощность.|Интегральное насыщение → резкий перегрев и аварийное отключение.|
|**Kd** (Дифференциальная)|Реагирует на **скорость изменения** температуры. Предсказывает будущее.|Если температура быстро растёт и почти достигла уставки, Kd заранее сбрасывает мощность, чтобы не было инерционного перелёта.|При шуме датчика реле будет хаотично щёлкать.|

**Формула внутри `PID_v1.h`:**
```cpp
output = Kp * error + Ki * ∫error*dt + Kd * (error - prevError)/dt
```
В скетче результат всегда приводится к диапазону **0…255**.

---

## 💻 Как это реализовано в коде

```cpp
// 1. Инициализация (в setup)
PID myPID(&input, &output, &setpoint, Kp, Ki, Kd, DIRECT);
myPID.SetMode(AUTOMATIC);        // PID включён
myPID.SetOutputLimits(0, 255);   // Выход от 0 до 255
myPID.SetSampleTime(1000);       // Расчёт раз в 1 секунду

// 2. Вычисление (в loop, ~каждую секунду)
myPID.Compute();                 // ← здесь магия PID
controlSSR(output);              // ← превращаем 0-255 в импульсы для реле
```
`myPID.Compute()` автоматически считает P, I и D, суммирует их, ограничивает результат 0-255 и записывает в переменную `output`.

## 🔌 Магия управления реле: Time Proportioning

ТЭН нельзя плавно менять по напряжению (он либо греет, либо нет). PID решает это **широтно-импульсным модулированием с окном 2 секунды**:
```cpp
void controlSSR(double pidOut) {
  static unsigned long winStart = 0;
  const unsigned long winSize = 2000; // 🔥 Окно 2 секунды
  unsigned long now = millis();
  if (now - winStart >= winSize) winStart = now; // Сброс окна
  
  // pidOut=0 → duty=0ms (выкл), pidOut=255 → duty=2000ms (вкл)
  unsigned long duty = (unsigned long)((pidOut / 255.0) * winSize);
  digitalWrite(SSR_PIN, (now - winStart < duty) ? HIGH : LOW);
}
```
📊 **Примеры:**

|`output` (из PID)|Duty (вкл за 2 сек)|Что делает ТЭН|
|---|---|---|
|`0`|`0 мс`|Полностью выключен|
|`128`|`1000 мс`|1 сек вкл / 1 сек выкл (50% мощности)|
|`255`|`2000 мс`|Постоянно включен|

## 🎛️ Как правильно настраивать Kp, Ki, Kd

В вашем скетче по умолчанию: `Kp=50.0, Ki=0.1, Kd=10.0`. Это стартовая точка для инерционных печей.

|Симптом|Что менять|Как менять|
|---|---|---|
|📉 Температура долго не доходит до уставки|Увеличить `Ki`|+0.05 за раз|
|📈 Постоянные скачки ±20-30°C вокруг уставки|Уменьшить `Kp`|-5 за раз|
|🔥 Резкий перелёт на 50-100°C выше уставки|Увеличить `Kd` или уменьшить `Kp`|+2 к `Kd` или -5 к `Kp`|
|⚡ Реле щёлкает каждые 0.5 сек при стабильной температуре|Уменьшить `Kd` или увеличить `Kp`|-2 к `Kd`|

💡 **Совет по настройке:**

1. Поставьте `Ki=0, Kd=0`. Поднимайте `Kp`, пока не появятся лёгкие колебания.
2. Верните `Kp` на 20% меньше.
3. Медленно добавляйте `Ki`, пока "хвост" недогрева не исчезнет.
4. Добавляйте `Kd`, чтобы сгладить первые перелёты при нагреве.

## ✅ Итог: цепочка работы PID в вашей печи
```
Датчик MAX31855 → input (текущая °C)
       ↓
PID_v1.Compute() → сравнивает с setpoint → считает P+I+D → выдаёт output (0-255)
       ↓
controlSSR(output) → превращает 0-255 в долю включения за 2 секунды
       ↓
SSR реле → ТЭН получает нужную среднюю мощность
       ↓
Температура плавно стабилизируется → input обновляется → цикл повторяется
```
