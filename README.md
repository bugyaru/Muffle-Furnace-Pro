## 📋 Обзор проекта **Muffle-Furnace-Pro**

Это **программируемый контроллер муфельной печи** на базе ESP32-C3 с веб-интерфейсом и PID-регулированием.

### 🔧 Основные компоненты:

|Компонент|Описание|
|---|---|
|**Микроконтроллер**|ESP32-C3 (единый мозг системы)|
|**Датчик температуры**|MAX31855 (термопара K-типа) через SPI|
|**Управление нагревом**|Твердотельное реле (SSR) с ШИМ|
|**Хранилище**|LittleFS (веб-файлы, программы, логи)|
|**Интерфейс**|Веб-сервер + WebSocket + Chart.js|

---

### 🌟 Ключевые функции:

1. **📡 Сетевые режимы**:
  - Точка доступа `Furnace_Config`
  - Подключение к роутеру (STA)
2. **🌡️ PID-регулирование**:
  - Алгоритм с настраиваемыми коэффициентами (Kp, Ki, Kd)
  - Time Proportioning: окно 2 секунды для управления SSR
  - Выход 0-255 → преобразуется в % включения реле
3. **📊 Веб-интерфейс**:
  - График температуры в реальном времени (обновление каждые 2 сек)
  - Ручное управление уставкой
  - Загрузка программ нагрева
  - API для интеграции
4. **📜 Программируемые профили**:
  - Многоступенчатые программы (температура, время)
  - Линейная интерполяция между точками
  - Сохранение в `programs.json`
5. **🛡️ Аварийная защита**:
  - Отключение при >1050°C
  - Автоматическое срабатывание при обрыве датчика (возврат 1050.1°C)
  - Независимо от режима работы

---

### 💻 Технические детали:

**Основной цикл (каждую секунду)**:
```
1. Чтение температуры → MAX31855
2. PID-расчёт → Compute()
3. Управление реле → controlSSR()
```

- **Kp** (пропорциональная) → реакция на текущую ошибку
- **Ki** (интегральная) → устраняет недогрев со временем
- **Kd** (дифференциальная) → предотвращает перелёт

## 🧠 Общая архитектура

ESP32-C3 выступает **единым мозгом** системы:

- 📡 **Сеть**: работает в режиме точки доступа (`Furnace_Config`) или подключается к вашему роутеру (STA).
- 🌐 **Веб-сервер**: отдаёт `index.html`, `chart.js`, CSS и принимает API-запросы.
- 🔄 **WebSocket**: отправляет данные в браузер каждые 2 секунды без перезагрузки страницы.
- 🌡️ **Датчик**: MAX31855 читает термопару по SPI.
- ⚡ **Управление**: PID-регулятор рассчитывает мощность, а `controlSSR()` включает/выключает твердотельное реле.
- 💾 **Хранение**: LittleFS хранит веб-файлы, программы нагрева и логи.

## 🔄 Как работает основной цикл (`loop()`)

Каждые `1000 мс` (1 сек) происходит следующее:

1. **Чтение температуры** → `readTemperature()`
2. **PID-расчёт** → `myPID.Compute()` (если включён нагрев)
3. **Управление реле** → `controlSSR()` (включает SSR на нужную часть 2-секундного окна)
4. **Аварийная проверка** → если `input > 1050.0` → мгновенное отключение SSR
5. **Логирование** → запись в CSV каждые 10 сек
6. **Отправка в браузер** → `sendWsStatus()` каждые 2 сек через WebSocket
7. **Обновление OLED** → каждые 500 мс
8. `server.handleClient()` и `webSocket.loop()` обрабатывают входящие HTTP/WS запросы

## 📡 Веб-сервер и WebSocket: как они общаются

|Действие пользователя|Что происходит в ESP32|
|---|---|
|Открывает `192.168.1.127`|`handleFileRequest()` находит `/index.html`, вызывает `server.streamFile()` (она сама ставит заголовки и отдаёт файл)|

|Добавляет программу|`POST /api/programs` → валидирует формат, обновляет массив, сохраняет в `/programs.json`|

## 🌡️ Чтение датчика и заглушка `1050.1°C`

```cpp
float readTemperature() {
  float t = thermocouple.readCelsius();
  if (isnan(t) || t > 1200 || t < -100) {
    // Если обрыв, замыкание или нет ответа → аварийная температура
    input = 1050.1; 
    return input;
  }
  input = t;
  return input;
}
```
- `1050.1` выбрано специально: оно **строго больше** `EMERGENCY_TEMP (1050.0)`, поэтому условие `if (input > EMERGENCY_TEMP)` мгновенно срабатывает и отключает нагрев.
- Это безопасный fallback: даже если датчик отвалится, печь не уйдёт в перегрев.

---

## ⚙️ PID и управление нагревом

- `myPID.Compute()` вычисляет `output` от `0` до `255`.
- `controlSSR()` преобразует это в **широтно-импульсную модуляцию** с окном `2000 мс`:
    - Если `output = 128` (50%) → SSR включён 1 сек, выключен 1 сек.
    - Если `output = 0` → SSR всегда выключен.
    - Если `output = 255` → SSR всегда включён.

---

## 📜 Как работают программы нагрева

1. Файл `programs.json` хранит массив: `[{"name":"Отжиг","prg":"600,30;850,45"}]`
2. `parseProgramString()` разбивает строку на шаги: `temp → 600, min → 30`
3. `runProgramStep()` линейно интерполирует уставку между шагами
```cpp
setpoint = prevTemp + (targetTemp - prevTemp) * progress
```
1. После завершения всех шагов → программа останавливается, нагрев отключается.

## 🔍 Что такое PID в вашей печи?

PID (**P**roportional-**I**ntegral-**D**erivative) — это алгоритм, который **рассчитывает нужную мощность нагрева**, чтобы температура плавно достигла заданной уставки (`setpoint`) и держалась на ней без раскачки и перелётов.

В вашем скетче он не управляет температурой напрямую, а управляет **скважностью включения ТЭНа** через твердотельное реле (SSR).

---

## ⚙️ Как работают три компоненты

|Компонент|За что отвечает|Пример в печи|Что будет, если параметр слишком большой|
|---|---|---|---|
|**Kp** (Пропорциональная)|Реагирует на **текущую ошибку**: `ошибка = setpoint - input`|Если до цели 200°C, даёт полную мощность. Если осталось 10°C, снижает мощность пропорционально.|Печь будет постоянно колебаться вокруг уставки (перелёт → недолёт).|
|**Ki** (Интегральная)|Накапливает ошибку **во времени**. Устраняет "хвост" недогрева.|Если печь долго не может дойти до 900°C (теряет тепло), Ki постепенно добавляет мощность.|Интегральное насыщение → резкий перегрев и аварийное отключение.|
|**Kd** (Дифференциальная)|Реагирует на **скорость изменения** температуры. Предсказывает будущее.|Если температура быстро растёт и почти достигла уставки, Kd заранее сбрасывает мощность, чтобы не было инерционного перелёта.|При шуме датчика реле будет хаотично щёлкать.|

**Формула внутри `PID_v1.h`:**
```cpp
output = Kp * error + Ki * ∫error*dt + Kd * (error - prevError)/dt
```
В скетче результат всегда приводится к диапазону **0…255**.

---

## 💻 Как это реализовано в коде

```cpp
// 1. Инициализация (в setup)
PID myPID(&input, &output, &setpoint, Kp, Ki, Kd, DIRECT);
myPID.SetMode(AUTOMATIC);        // PID включён
myPID.SetOutputLimits(0, 255);   // Выход от 0 до 255
myPID.SetSampleTime(1000);       // Расчёт раз в 1 секунду

// 2. Вычисление (в loop, ~каждую секунду)
myPID.Compute();                 // ← здесь магия PID
controlSSR(output);              // ← превращаем 0-255 в импульсы для реле
```
`myPID.Compute()` автоматически считает P, I и D, суммирует их, ограничивает результат 0-255 и записывает в переменную `output`.

## 🔌 Магия управления реле: Time Proportioning

ТЭН нельзя плавно менять по напряжению (он либо греет, либо нет). PID решает это **широтно-импульсным модулированием с окном 2 секунды**:
```cpp
void controlSSR(double pidOut) {
  static unsigned long winStart = 0;
  const unsigned long winSize = 2000; // 🔥 Окно 2 секунды
  unsigned long now = millis();
  if (now - winStart >= winSize) winStart = now; // Сброс окна
  
  // pidOut=0 → duty=0ms (выкл), pidOut=255 → duty=2000ms (вкл)
  unsigned long duty = (unsigned long)((pidOut / 255.0) * winSize);
  digitalWrite(SSR_PIN, (now - winStart < duty) ? HIGH : LOW);
}
```
📊 **Примеры:**

|`output` (из PID)|Duty (вкл за 2 сек)|Что делает ТЭН|
|---|---|---|
|`0`|`0 мс`|Полностью выключен|
|`128`|`1000 мс`|1 сек вкл / 1 сек выкл (50% мощности)|
|`255`|`2000 мс`|Постоянно включен|

## 🎛️ Как правильно настраивать Kp, Ki, Kd

В вашем скетче по умолчанию: `Kp=50.0, Ki=0.1, Kd=10.0`. Это стартовая точка для инерционных печей.

|Симптом|Что менять|Как менять|
|---|---|---|
|📉 Температура долго не доходит до уставки|Увеличить `Ki`|+0.05 за раз|
|📈 Постоянные скачки ±20-30°C вокруг уставки|Уменьшить `Kp`|-5 за раз|
|🔥 Резкий перелёт на 50-100°C выше уставки|Увеличить `Kd` или уменьшить `Kp`|+2 к `Kd` или -5 к `Kp`|
|⚡ Реле щёлкает каждые 0.5 сек при стабильной температуре|Уменьшить `Kd` или увеличить `Kp`|-2 к `Kd`|

💡 **Совет по настройке:**

1. Поставьте `Ki=0, Kd=0`. Поднимайте `Kp`, пока не появятся лёгкие колебания.
2. Верните `Kp` на 20% меньше.
3. Медленно добавляйте `Ki`, пока "хвост" недогрева не исчезнет.
4. Добавляйте `Kd`, чтобы сгладить первые перелёты при нагреве.

```
Датчик MAX31855 → input (текущая °C)
       ↓
PID_v1.Compute() → сравнивает с setpoint → считает P+I+D → выдаёт output (0-255)
       ↓

       ↓
Температура плавно стабилизируется → input обновляется → цикл повторяется
```

## 🔌 Схема подключения

### Необходимые компоненты:
- ESP32-C3 Dev Module
- MAX31855 (модуль для термопары K-типа)
- Твердотельное реле (SSR) 5V
- Термопара K-типа
- Блок питания 5V для ESP32 и SSR
- Резистор 220Ω (опционально, для индикации)

### Распиновка ESP32-C3 → MAX31855 (SPI)

|MAX31855|ESP32-C3|Примечание|
|---|---|---|
|VCC|3.3V|Питание датчика|
|GND|GND|Общая земля|
|DO|GPIO4 (MISO)|Данные от датчика|
|CS|GPIO5 (SS)|Выбор устройства|
|CLK|GPIO6 (SCK)|Тактовый сигнал|

> ⚠️ **Важно:** MAX31855 работает только на 3.3V! Не подключайте к 5V.

### Распиновка ESP32-C3 → SSR

|SSR|ESP32-C3|Примечание|
|---|---|---|
|+ (Signal)|GPIO7|Управление реле|
|- (GND)|GND|Общая земля|
|AC Input|~220V|Фаза от сети|
|AC Output|ТЭН|Нагрузка (нагреватель)|

> ⚠️ **Осторожно!** Подключение к сети 220V требует соблюдения техники безопасности. Используйте предохранитель и правильную изоляцию.

### Полная схема подключения

```
                    +---------------------+
                    |     ESP32-C3        |
                    |                     |
   [Термопара K]----| MAX31855 (SPI)      |
                    | DO  → GPIO4         |
                    | CS  → GPIO5         |
                    | CLK → GPIO6         |
                    |                     |
                    | GPIO7 ──[220Ω]──> + | SSR
                    | GND  ───────────> - | SSR
                    |                     |
                    +----------+----------+
                               |
                    AC 220V ---+---[SSR]---[ТЭН]--- N
                               |
                            (Фаза)
```

### Рекомендации по монтажу:
1. **Максимальная длина проводов** от термопары до MAX31855 — не более 1 метра
2. **Экранирование**: используйте экранированные провода для SPI-линий
3. **Заземление**: все компоненты должны иметь общую землю
4. **Теплоотвод**: установите SSR на радиатор при мощности >1kW
5. **Предохранитель**: добавьте автоматический выключатель 10-16A в цепь 220V

## 🐛 Troubleshooting (Диагностика проблем)

### Проблема: Датчик показывает 1050.1°C или ошибку

**Возможные причины:**
- Обрыв термопары или плохой контакт
- Неправильное подключение SPI
- Недостаточное питание MAX31855

**Решение:**
```cpp
// Проверьте в коде:
Serial.println(thermocouple.readCelsius());
// Если возвращается NAN или >1200 — проблема с подключением
```

**Шаги диагностики:**
1. Проверьте пайку контактов MAX31855
2. Убедитесь, что VCC = 3.3V (не 5V!)
3. Проверьте целостность термопары мультиметром
4. Попробуйте другой модуль MAX31855

---

### Проблема: Печь не нагревается, реле не включается

**Возможные причины:**
- Неверный GPIO для SSR
- SSR неисправен
- PID выдает output = 0
- Аварийная защита активирована

**Решение:**
```cpp
// В loop() добавьте отладку:
Serial.printf("Output: %.1f, SSR State: %d\n", output, digitalRead(SSR_PIN));
```

**Шаги диагностики:**
1. Проверьте `SSR_PIN` в коде (должен быть GPIO7)
2. Подайте 5V напрямую на SSR — должен щёлкнуть
3. Проверьте значение `output` через веб-интерфейс
4. Убедитесь, что температура < 1050°C (нет аварии)

---

### Проблема: Температура колеблется ±30-50°C

**Возможные причины:**
- Неправильные коэффициенты PID
- Слишком большое окно ШИМ
- Инерционность печи не учтена

**Решение:**
- Уменьшите `Kp` на 5-10 единиц
- Увеличьте `Kd` на 2-5 единиц
- Проверьте `Ki` — если слишком большой, уменьшите

**Быстрая настройка:**
1. Установите `Kp=30, Ki=0, Kd=0`
2. Запустите нагрев до 500°C
3. Если есть перелёт — уменьшайте `Kp`
4. Если не доходит — добавляйте `Ki` по 0.05
5. Для сглаживания добавьте `Kd=5-15`

---

### Проблема: Веб-интерфейс не загружается

**Возможные причины:**
- Файлы LittleFS не загружены
- Неправильная partition scheme
- Ошибка компиляции

**Решение:**
1. Проверьте наличие папки `data/` со скетчем
2. Убедитесь, что `index.html`, `chart.js`, `styles.css` в папке `data/`
3. Загрузите файловую систему: `Tools → ESP32 Sketch Data Upload`
4. Проверьте Partition Scheme: `Custom` с `partitions.csv`

**Проверка в коде:**
```cpp
if (!LittleFS.begin(true)) {
  Serial.println("LittleFS mount failed!");
  return;
}
```

### Проблема: WebSocket не обновляет данные

**Возможные причины:**
- Браузер заблокировал соединение
- Ошибка в JavaScript
- Переполнение буфера

**Решение:**
1. Откройте консоль браузера (F12) — проверьте ошибки
2. Обновите страницу с очисткой кэша (Ctrl+F5)
3. Проверьте лог ESP32 через Serial Monitor
4. Уменьшите частоту отправки данных (сейчас 2 сек)

---

### Проблема: Программы нагрева не запускаются

**Возможные причины:**
- Неправильный формат `programs.json`
- Ошибка парсинга строки программы
- Конфликт с ручным режимом

**Решение:**
Проверьте формат JSON:
```json
[
  {"name":"Отжиг","prg":"600,30;850,45;500,60"}
]
```
- Температура в °C, время в минутах
- Разделитель шагов — точка с запятой `;`
- Разделитель параметров — запятая `,`

---

### Таблица быстрых решений

|Симптом|Быстрое решение|
|---|---|
|❌ 1050.1°C|Проверить контакты MAX31855|
|❌ Реле молчит|Проверить GPIO7 и питание SSR|
|📉 Долгий нагрев|Увеличить `Ki` на 0.05|
|📈 Перегрев|Уменьшить `Kp` на 5, увеличить `Kd` на 2|
|🌐 Нет веба|Перезагрузить LittleFS|
|📜 Программа не работает|Проверить формат JSON|

## 🌐 API-документация

Контроллер предоставляет REST API и WebSocket для интеграции с внешними системами.

### Базовый URL
```
http://192.168.1.127/api
```
*(IP может отличаться в зависимости от настроек сети)*

---

### HTTP Endpoints

#### GET `/api/status`
Получить текущее состояние системы.

**Ответ:**
```json
{
  "temp": 245.7,
  "setpoint": 500.0,
  "output": 65.3,
  "heatingEnabled": true,
  "programRunning": false,
  "emergencyActive": false
}
```

**Пример запроса (curl):**
```bash
curl http://192.168.1.xxx/api/status
```

---

#### POST `/api/control`
Изменить уставку температуры или включить/выключить нагрев.

**Запрос:**
```json
{
  "setpoint": 600.0,
  "heatingEnabled": true
}
```

**Пример (curl):**
```bash
curl -X POST http://192.168.1.xxx/api/control \
  -H "Content-Type: application/json" \
  -d '{"setpoint":600.0,"heatingEnabled":true}'
```

**Ответ:**
```json
{"status":"ok","message":"Control updated"}
```

---

#### POST `/api/pid`
Настроить коэффициенты PID.

**Запрос:**
```json
{
  "kp": 50.0,
  "ki": 0.1,
  "kd": 10.0
}
```

**Пример (curl):**
```bash
curl -X POST http://192.168.1.xxx/api/pid \
  -H "Content-Type: application/json" \
  -d '{"kp":45.0,"ki":0.15,"kd":12.0}'
```

---

#### GET `/api/programs`
Получить список сохранённых программ нагрева.

**Ответ:**
```json
[
  {"name":"Отжиг","prg":"600,30;850,45;500,60"},
  {"name":"Закалка","prg":"900,60;200,120"}
]
```

---

#### POST `/api/loadProgram`
Запустить программу нагрева по имени.

**Запрос:**
```json
{
  "programName": "Отжиг"
}
```

**Пример (curl):**
```bash
curl -X POST http://192.168.1.xxx/api/loadProgram \
  -H "Content-Type: application/json" \
  -d '{"programName":"Отжиг"}'
```

**Ответ:**
```json
{"status":"ok","message":"Program started"}
```

---

#### POST `/api/programs`
Добавить или обновить программу в хранилище.

**Запрос:**
```json
{
  "name": "Новая программа",
  "prg": "400,20;700,40;300,30"
}
```

**Пример (curl):**
```bash
curl -X POST http://192.168.1.xxx/api/programs \
  -H "Content-Type: application/json" \
  -d '{"name":"Тест","prg":"500,15;800,30"}'
```

---

#### DELETE `/api/programs/{name}`
Удалить программу по имени.

**Пример (curl):**
```bash
curl -X DELETE http://192.168.1.xxx/api/programs/Тест
```

---

#### GET `/api/log`
Получить последние записи из температурного лога.

**Параметры:**
- `lines` (опционально): количество строк (по умолчанию 100)

**Ответ:**
```csv
timestamp,temp,setpoint,output
1699000000,24.5,0.0,0.0
1699000010,125.3,500.0,85.2
1699000020,245.7,500.0,65.3
```

### WebSocket API

Подключение: `ws://192.168.1.xxx/ws`

#### Сообщения от сервера (ESP32 → Клиент)

Сервер отправляет JSON каждые 2 секунды:

```json
{
  "temp": 245.7,
  "setpoint": 500.0,
  "output": 65.3,
  "heatingEnabled": true,
  "programRunning": false,
  "emergencyActive": false,
  "tempHistory": [
    {"time":"10:00:00","temp":24.5},
    {"time":"10:00:02","temp":125.3},
    {"time":"10:00:04","temp":245.7}
  ]
}
```

#### Сообщения от клиента (Клиент → ESP32)

Клиент может отправлять команды:

**Изменить уставку:**
```json
{"command":"setSetpoint","value":600.0}
```

**Включить/выключить нагрев:**
```json
{"command":"toggleHeating","enabled":true}
```

**Запустить программу:**
```json
{"command":"startProgram","name":"Отжиг"}
```

**Пример подключения через JavaScript:**
```javascript
const ws = new WebSocket('ws://192.168.1.127/ws');

ws.onopen = () => {
  console.log('Connected to furnace');
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Temperature:', data.temp);
};

// Отправка команды
ws.send(JSON.stringify({
  command: 'setSetpoint',
  value: 700.0
}));
```

## 📋 Примеры программ нагрева

### Пример 1: Отжиг стали (мягкий отпуск)

**Цель:** Снять внутренние напряжения, улучшить обрабатываемость.

```json
{
  "name": "Отжиг стали 45",
  "prg": "550,60;850,90;600,120;200,180"
}
```

**Расшифровка:**
1. Нагрев до 550°C за 60 минут (предварительный подогрев)
2. Нагрев до 850°C за 90 минут (аустенизация)
3. Медленное охлаждение до 600°C за 120 минут (изотермическая выдержка)
4. Охлаждение до 200°C за 180 минут (печное охлаждение)

**График:**
```
Температура (°C)
    |
850 |           ●━━━━━━━━━●
    |          /           \
600 |         /             ●━━━━
    |        /                      \
550 |   ●━━━●                        \
    |  /                              \
200 | ●                                ●━━━━
    +----------------------------------------→ Время (мин)
      0   60   150   270              450
```

---

### Пример 2: Закалка инструментальной стали

**Цель:** Получить высокую твёрдость и износостойкость.

```json
{
  "name": "Закалка У8",
  "prg": "400,30;780,45;200,15"
}
```

**Расшифровка:**
1. Предварительный подогрев до 400°C за 30 минут (для избежания трещин)
2. Нагрев до 780°C за 45 минут (закалочная температура)
3. Быстрое охлаждение до 200°C за 15 минут (перед закалкой в масло)

> ⚠️ **Внимание:** После достижения 200°C деталь извлекается и закаливается в масле вручную!

---

### Пример 3: Обжиг керамики

**Цель:** Удалить влагу, подготовить к глазурированию.

```json
{
  "name": "Бисквитный обжиг",
  "prg": "150,60;350,90;600,60;950,120;100,240"
}
```

**Расшифровка:**
1. Сушка при 150°C за 60 минут (удаление свободной воды)
2. Нагрев до 350°C за 90 минут (удаление химически связанной воды)
3. Нагрев до 600°C за 60 минут (выгорание органики)
4. Нагрев до 950°C за 120 минут (спекание)
5. Медленное охлаждение до 100°C за 240 минут (предотвращение трещин)

**Рекомендации:**
- Общая длительность: ~9 часов
- Критически важно медленное охлаждение!

---

### Пример 4: Старение алюминиевых сплавов

**Цель:** Повысить прочность после закалки.

```json
{
  "name": "Старение Д16Т",
  "prg": "190,180"
}
```

**Расшифровка:**
1. Нагрев до 190°C и выдержка 180 минут (3 часа)
2. Естественное охлаждение в печи

**Особенности:**
- Требуется высокая точность: ±5°C
- Рекомендуется `Kp=30, Ki=0.05, Kd=5`

---

### Пример 5: Плавка цветных металлов (олово, свинец)

**Цель:** Расплавить металл для литья.

```json
{
  "name": "Плавка олова",
  "prg": "150,30;250,20;300,10"
}
```

**Расшифровка:**
1. Предварительный нагрев до 150°C за 30 минут
2. Нагрев до 250°C за 20 минут (температура плавления олова ~232°C)
3. Перегрев до 300°C за 10 минут (для лучшей текучести)

**Температуры плавления:**
- Олово (Sn): 232°C
- Свинец (Pb): 327°C
- Цинк (Zn): 420°C
- Алюминий (Al): 660°C

---

### Пример 6: Многоступенчатый сложный профиль

**Цель:** Термическая обработка биметаллических деталей.

```json
{
  "name": "Биметалл",
  "prg": "200,30;450,60;650,45;450,90;300,60;150,120"
}
```

**Расшифровка:**
1. 200°C за 30 мин — предварительный нагрев
2. 450°C за 60 мин — первая ступень
3. 650°C за 45 мин — пиковая температура
4. 450°C за 90 мин — изотермическая выдержка
5. 300°C за 60 мин — контролируемое охлаждение
6. 150°C за 120 мин — финальное охлаждение

### Как создать свою программу

**Шаг 1:** Определите температурные этапы
- Максимальная температура
- Время выдержки на каждом этапе
- Скорость нагрева/охлаждения

**Шаг 2:** Составьте строку программы
Формат: `"temp1,min1;temp2,min2;..."`

**Пример расчёта:**
- Нужно нагреть от 20°C до 600°C за 2 часа (120 мин)
- Средняя скорость: (600-20)/120 = 4.8°C/мин
- Разбейте на этапы: `200,30;400,60;600,30`

**Шаг 3:** Добавьте через API или веб-интерфейс
```bash
curl -X POST http://192.168.1.xxx/api/programs \
  -H "Content-Type: application/json" \
  -d '{"name":"Моя программа","prg":"200,30;400,60;600,30"}'
```

**Шаг 4:** Проверьте в `programs.json`
```json
[
  {"name":"Моя программа","prg":"200,30;400,60;600,30"}
]
```

### Советы по составлению программ

✅ **Делайте:**
- Добавляйте предварительный подогрев для массивных деталей
- Учитывайте инерционность печи (температура растёт с задержкой)
- Используйте медленное охлаждение для предотвращения трещин
- Тестируйте программы на пустой печи перед загрузкой деталей

❌ **Не делайте:**
- Резкие скачки температуры (>10°C/мин без необходимости)
- Превышение максимальной температуры печи (1050°C)
- Игнорирование времени выдержки (химические процессы требуют времени)
- Запуск программ без проверки крепления термопары
